# BalaLive Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Steamodded Balatro mod that serves a localhost OBS overlay showing localized jokers, consumables, and poker hand levels.

**Architecture:** `src/state.lua` is pure and testable. `src/server.lua` owns nonblocking LuaSocket HTTP/SSE behavior. `main.lua` wires Steamodded config, state snapshots, the server, and the Game update hook. `web/` contains static frontend files served by the mod.

**Tech Stack:** Lua/LuaJIT, Steamodded, LuaSocket, vanilla browser `EventSource`, HTML/CSS/JS.

---

## File Structure

- Create `metadata.json`: Steamodded metadata with author `ChromaPIE`.
- Create `config.lua`: default persisted config.
- Create `main.lua`: Steamodded entrypoint, config normalization, server startup, update hook.
- Create `src/state.lua`: localized snapshot building, duplicate merge, rarity colors, signature, JSON encoding.
- Create `src/server.lua`: nonblocking localhost HTTP/SSE server.
- Create `src/config_ui.lua`: Steamodded config tab nodes.
- Create `web/index.html`: transparent OBS overlay root.
- Create `web/style.css`: compact overlay styling, rarity text/background variants, animations.
- Create `web/app.js`: EventSource client, panel rotation, DOM diff animations.
- Create `tests/state_spec.lua`: state module tests.
- Create `tests/server_spec.lua`: server formatting/route tests.
